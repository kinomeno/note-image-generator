# アプリ仕様書 — ととのえ

最終更新：2026-05-24 / 対象バージョン：v0.15

---

## 1. 概要
- **目的**：任意の画像を note 推奨サイズ（1920×1006px）等へ、モチーフを中央配置して自動整形する。
- **構成**：
  - **Web版**（`docs/index.html`）：単一HTML完結。GitHub Pages で公開。1枚特化・無料＋パスワード解放。
  - **デスクトップ版**（`desktop/note画像生成アプリ.py`）：Python/Tkinter。フォルダ一括・高機能。

## 3. Web版 仕様

### 3-1. 技術
- 依存ゼロの単一HTML（CSS/JSインライン）。
- ZIP生成時のみ JSZip(3.10.1) を CDN から動的ロード。
- 画像処理は Canvas 2D。ハッシュは Web Crypto（`crypto.subtle`）。

### 3-2. 主要定数
| 定数 | 値 | 意味 |
|---|---|---|
| `FILL_W` | 0.58 | モチーフ幅のキャンバス幅に対する基準比 |
| `FILL_H` | 0.71 | モチーフ高のキャンバス高に対する基準比 |
| `WHITE_THRESH` | 240 | 「白に近い」と判定するしきい値（各チャンネル） |
| `PAD` | 60 | バウンディングボックス外側の余白(px) |

### 3-3. サイズプリセット
`note(1920×1006) / x(1024×512) / instagram(1080×1080) / youtube(1280×720)` ＋ `custom(任意)`。

### 3-4. 変換アルゴリズム `processToCanvas(img, W, H, bg, scaleMul)`
1. 元画像を `bg` で塗ったキャンバスに描画（透過の下地を確定）。
2. 1/3 に縮小し `getImageData`。各チャンネルいずれかが `WHITE_THRESH` 未満のピクセル範囲を走査してバウンディングボックスを取得。
3. 非白が無ければ画像全体をモチーフ扱い。あれば `PAD` を足して（×3 で原寸換算）切り出し。
4. `scale = min(W*FILL_W/cw, H*FILL_H/ch) * scaleMul`。`nw>W` / `nh>H` を超えないようクランプ。
5. `W×H` のキャンバスを `bg` で塗り、モチーフを中央へ `drawImage`。

### 3-5. サイズスライダー
- `scaleMul` を 0.5〜1.6（UI表示 50〜160%）で可変。無料・常時表示。

### 3-5a2. 位置調整（v0.13〜・追加要素）
- `state.offsetX / offsetY`（キャンバス比 −0.5〜0.5）。貼付座標 = 中央 ±オフセット。`px/py` はキャンバス内に収まるようクランプ。
- UI：こまかい設定の「位置（横／縦）」スライダー＋中央リセット。

### 3-5a3. ジェスチャ操作（v0.14〜）
- プレビュー canvas に `touch-action:none`。
- 2本指ピンチ → `setSizeScale()`（無料）。1本指ドラッグ → `setOffset()`（解放後のみ）。デスクトップはマウスドラッグで `setOffset()`（解放後）。
- スライダーとジェスチャは `setSizeScale()/setOffset()` 経由で値・UIを同期。

### 3-5b. テキスト追加（v0.11〜）
- `drawText(ctx,W,H,bg)` を変換の最後に呼ぶ。`state.text` が空なら何もしない。
- フォント：`textFont`（gothic/mincho）。サイズ：`textSizeFactor`（H比、既定0.075）。
- 位置：`textPos`（top/middle/bottom、既定bottom）。
- 色：`textColor`（"auto"＝背景輝度で白/濃色を自動選択、または指定HEX）。視認性のため縁取り（stroke）を併用。
- 無料：入力欄のみ（位置=下・色=自動・既定サイズ/ゴシック固定）。解放後：位置・サイズ・色・フォントを変更可。

### 3-6. 追加要素（パスワード解放）
- 入力文字列を SHA-256 し、埋め込みハッシュ `UNLOCK_HASH` と比較。
  - v0.10 のコード：`ttnoe` / ハッシュ：`2ff356f2ce3da91cbaf702ab1e26f1f0253a07a301bc106dcebde5ac6825f250`
- 成功時 `localStorage['ttnoe_premium']='1'`。以後は自動で解放。
- **注**：クライアント完結のソフトゲート（ソース解析で突破可能）。note有料記事の「合言葉」程度の位置づけ。
- コード変更手順：新コードの SHA-256 を求め、`UNLOCK_HASH` を差し替える。

### 3-7. 状態の永続化（localStorage）
| キー | 値 | 用途 |
|---|---|---|
| `ttnoe_premium` | "1" | 解放状態 |
| `ttnoe_detail_open` | "1"/"0" | こまかい設定の開閉 |

### 3-8. ダウンロード
- 単一サイズ：`canvas.toBlob()` で直接保存。
- 複数サイズ：JSZip で `元名_サイズ.拡張子` を束ねて `元名_ととのえ.zip`。
- JPG/WebP は品質 0.95。

### 3-9. レスポンシブ
- ブレークポイント `max-width:760px`。
- PC：`body{height:100vh; overflow:hidden}`、プレビューは `max-height:42vh` 等で縦スクロールを抑止。
- モバイル：`overflow-x:hidden; overflow-y:auto`（縦スクロール可・横不可）。

### 3-10. デザイントークン
- 差し色（アクセント）：`--accent:#5f8276`（くすみセージ）。
- ロゴ：明朝系。文字色 `#1d1d22`、サブ `#5f5f68`。

### 3-11. テスト用クエリ／フラグ
- `?lock=1`：起動時に `ttnoe_premium` を削除し、ロック状態で表示。
- 画面上中央の「🧪」トグルで解放/未解放を即切替。
- **`const TEST_MODE`（v0.15〜）**：`false` で🧪トグルをDOMから除去（公開版）。`true` で開発用に復活。
- **公開版**：`公開版/index.html`（`TEST_MODE=false`）＋アセット一式を別途生成。開発の正本は `docs/index.html`。

### 3-14. OGP／ファビコン（v0.15〜）
- `docs/ogp.png`（1200×630）、`docs/favicon.svg` ＋ `docs/favicon.png`。
- `<head>` に OGP / Twitter カードのメタ。**公開時は `og:url` / `og:image` を配信先の絶対URLに合わせる**こと。

### 3-12. 多言語（日/英）— v0.12〜
- `navigator.language` から自動判定（`en*` → 英語、その他 → 日本語）。`localStorage['ttnoe_lang']` があれば優先。
- UI文言は `data-i18n / data-i18n-html / data-i18n-ph / data-i18n-title` 属性＋辞書 `I18N` で差し替え（`applyI18n()`）。
- 動的文言（プレビュー寸法・DLボタンtitle・アラート・エラー・テストトグル）は `t(key)` 経由。
- フッターの「EN / 日本語」ボタンで手動切替（`setLang()`）。

### 3-13. クレジット／支援
- フッターに作者「キノメノ」・紹介URL（note記事）・Ko-fi・note チップのリンクを表示。
- 紹介URL：`https://note.com/kinomeno/n/n67ed8c0ac3db` ／ Ko-fi：`https://ko-fi.com/kinomeno`

## 4. デスクトップ版 仕様

### 4-1. 技術
- Python 3.10+ / Tkinter + **tkinterdnd2** / **Pillow** / **numpy**。

### 4-2. 機能
- D&D、リアルタイムプレビュー、複数サイズ同時生成。
- フィルター：セピア／モノクロ／彩度／明度／コントラスト／ぼかし。
- テキスト追加（内容・サイズ・位置・自動シャドウ・日本語フォント探索）。
- 背景色、出力形式（PNG/JPG/WebP）。
- バッチ（複数フォルダのジョブ）＋スケジュール（毎日指定時刻）。
- プリセット（JSON）。保存先：`~/.note_image_generator/`（`presets.json` / `batch_jobs.json`）。

### 4-3. ビルド
- `build_exe.py` が PyInstaller を呼び出し：
  `--onefile -w --hidden-import=tkinterdnd2` → `desktop/dist/note画像生成アプリ.exe`。

## 5. 既知の制約 / 注意
- 自動検出は「白背景＋被写体」で最良。複雑背景・全面写真は画像全体配置になりやすい。
- Web版のパスワードは強固な認証ではない（ソフトゲート）。
- 複数サイズZIPは JSZip の CDN 取得が必要（オフライン不可）。
- `crypto.subtle` はセキュアコンテキスト必須（https / localhost / file:// は可）。
